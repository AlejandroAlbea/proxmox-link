Documentación para Cambiar el Puerto de Proxmox y Configurar Nginx

Esta guía detalla los pasos necesarios para cambiar el puerto de acceso de Proxmox y configurar Nginx como un proxy inverso. Esto te permitirá acceder a Proxmox a través de pve01.antfor.com.

Requisitos Previos

  • Acceso administrativo a tu servidor Proxmox.
  • Dominio pve01.antfor.com configurado para apuntar a la IP de tu servidor Proxmox.
  • Nginx instalado en el servidor o en otro servidor que pueda comunicarse con Proxmox.

Cambiar el Puerto de Proxmox

  1. Conectar al Servidor Proxmox:

    Conéctate a tu servidor Proxmox mediante SSH:

    ssh root@IP_DE_TU_PROXMOX

  2. Editar el Archivo de Configuración de Proxmox:

    Abre el archivo de configuración de Proxmox:

    nano /etc/default/pveproxy

    Busca la línea que contiene PVE_PORT y cambia el puerto a tu preferencia. Por ejemplo:

    PVE_PORT=8006

  3. Reiniciar el Servicio de Proxmox:

    Después de realizar el cambio, reinicia el servicio de Proxmox:

    systemctl restart pveproxy

Configurar Nginx como Proxy Inverso

  1. Instalar Nginx (si no está instalado):

    Si Nginx no está instalado, instálalo usando:

    apt update
apt install nginx

  2. Configurar Nginx:

    Crea un archivo de configuración para Proxmox:

    nano /etc/nginx/sites-available/proxmox

    Agrega la siguiente configuración, reemplazando IP_DE_TU_PROXMOX y el puerto que has configurado:

    server {
    listen 80;
    server_name proxmox.antfor.com;

    location / {
        proxy_pass https://IP_DE_TU_PROXMOX:8006;  # Cambia el puerto si lo has cambiado
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Opcional: Desactivar verificación de SSL en el proxy_pass
        proxy_ssl_verify off; 
    }
}

  3. Habilitar la Configuración:

    Crea un enlace simbólico para habilitar la configuración:

    ln -s /etc/nginx/sites-available/proxmox /etc/nginx/sites-enabled/

  4. Probar la Configuración de Nginx:

    Asegúrate de que no haya errores en la configuración de Nginx:

    nginx -t

  5. Reiniciar Nginx:

    Reinicia Nginx para aplicar la nueva configuración:

    systemctl restart nginx

Configurar Certificados SSL

Si quieres habilitar HTTPS en Nginx, necesitarás certificados SSL. Puedes usar Let's Encrypt para obtener certificados gratuitos.

  1. Instalar Certbot:

    Si no tienes Certbot, instálalo:

    apt install certbot python3-certbot-nginx

  2. Obtener Certificados:

    Ejecuta el siguiente comando para obtener certificados:

    certbot --nginx -d proxmox.antfor.com

    Sigue las instrucciones para completar el proceso de configuración.

  3. Configurar Nginx para Usar HTTPS:

    Si ya tienes certificados, puedes modificar la configuración de Nginx para redirigir el tráfico HTTP a HTTPS:

    Modifica el archivo /etc/nginx/sites-available/proxmox:

    server {
    listen 80;
    server_name proxmox.antfor.com;
    return 301 https://$host$request_uri;  # Redirigir a HTTPS
}

server {
    listen 443 ssl;
    server_name proxmox.antfor.com;

    ssl_certificate /etc/letsencrypt/live/proxmox.antfor.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/proxmox.antfor.com/privkey.pem;

    location / {
        proxy_pass https://IP_DE_TU_PROXMOX:8006;  # Cambia el puerto si lo has cambiado
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_ssl_verify off; 
    }
}

  4. Reiniciar Nginx:

    Vuelve a reiniciar Nginx:

    systemctl restart nginx

Acceso a Proxmox

Ahora deberías poder acceder a Proxmox mediante el dominio https://proxmox.antfor.com, usando el puerto configurado (8006 en este caso).

Notas Adicionales

  • Realiza copias de seguridad antes de realizar cambios importantes.
  • Revisa los registros de Nginx en caso de problemas: /var/log/nginx/error.log.

Problemas Comunes

  • Problemas de Conexión: Verifica que el puerto de Proxmox esté accesible y que Nginx esté configurado correctamente.
  • Errores de Certificado: Asegúrate de que los certificados SSL estén instalados y configurados correctamente.
